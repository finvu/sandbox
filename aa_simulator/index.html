    AA API Sandbox

    This is a Sandbox implementation of the AA API and can be used for testing the Account Aggregator flows as part of the ecosystem.

    Request headers

    Below http headers need to be set as part of each request

    Key Value
    content-Type application/xml
    fip_api_key Please email us on info@cookiejar.co.in for obtaining the key

    API Request

    The API URL to be used is <Root API Endpoint>/<FIP API> as per ReBIT AA API Specs.

    Consents

    This API is intended for AA Client to request generation of digitally signed consent artefacts.

    HTTP METHOD: POST

    URL: http://aa.finvu.in/API/V1/Consents

    Request Body

    <?xml version="1.0" encoding="UTF-8"?>
<ConsentsRequest xmlns="http://standards.rebit.org.in/aa" ver="1.0" timestamp="2018-07-07T06:49:32.606Z" txnid="4a4adbbe-29ae-11e8-a8d7-0289437bf331">
    <FIP id="fip-1"></FIP>
    <FIU id="fiu-1"></FIU>
    <AA id="aa-1"></AA>
    <Customer idTypes="string" id="string"></Customer>
    <FITypes>
        <FIType>DEPOSIT</FIType>
    </FITypes>
    <Data-Items>
        <Data id="string" type="TRANSACTIONAL">
            <Access mode="STORE"></Access>
            <DateTimeRange from="2017-07-13T11:33:34.509Z" to="2018-07-13T11:33:34.509Z"></DateTimeRange>
            <DataLife unit="MONTH" value="string"></DataLife>
            <Frequency unit="DAILY"></Frequency>
            <Datafilter></Datafilter>
        </Data>
    </Data-Items>
    <ConsentUse logUri="string"></ConsentUse>
    <DataAccess logUri="string"></DataAccess>
    <Purpose code="string" refUri="string" text="string">
        <Category type="string"></Category>
    </Purpose>
    <Signature>c3RyaW5n</Signature>
</ConsentsRequest>

    Response Expected: 

    <ConsentsResponse ver="1.0" timestamp="2018-07-07T06:49:32.606+0000" txnid="4a4adbbe-29ae-11e8-a8d7-0289437bf331" id="jayavants" type="USER">
    <ConsentsHandle>4e5b5961-2514-42e0-b3f6-b9cf8de6e6de</ConsentsHandle>
</ConsentsResponse>

    Consents/{consentsHandle}

    This API is intended for checking the status of a previously submitted Consent Artefacts creation request

    HTTP METHOD: GET

    URL: http://aa.finvu.in/API/V1/Consents/{consentsHandle}

    Request Body

    No Request Body expected

    Response Expected: 

    <ConsentStatus ver="1.0" timestamp="2018-07-07T06:49:32.606+0000" txnid="e9cc2222-d4bb-4eb1-9e1b-2222fbff4dry">
    <ConsentsHandle>5f211951-46a0-4c53-bba6-e60f851b7687</ConsentsHandle>
    <Status>READY</Status>
    <Consents>
        <Consent id="345678c8-29c8-11e8-8865-jhdr7lkg8745p" status="READY"/>
    </Consents>
</ConsentStatus>

    Notification

    API handles the notifications corresponding to the events generated during consent flow and data flow.

    HTTP METHOD: POST

    URL: http://aa.finvu.in/API/V1/Notification

    Request Body

    <?xml version="1.0" encoding="UTF-8"?>
<Notification xmlns="http://standards.rebit.org.in/aa" ver="1.0" timestamp="2018-07-06T04:37:19.509Z" txnid="0b811819-9044-4856-b0ee-8c88035f8858">
    <From type="string" value="string" name="SBI" id="fip123"></From>
    <To type="string" value="string" name="string" id="string"></To>
    <Event type="FI-READY"></Event>
    <Consent id="687c3620-6bb4-43c4-9b9e-39f36c7617fb" status="string"></Consent>
    <Session sessionId="9afdacdd-215b-4c17-af18-cdccebfb7d1a" status="string"></Session>
    <Payload>string</Payload>
</Notification>

    Response Expected: 

    200 OK

    Heartbeat

    This API is used by FIUs to check availability of AAs.

    HTTP METHOD: GET

    URL: http://aa.finvu.in/API/V1/Heartbeat

    Request Body

    No Request Body expected

    Response Expected: 

    <HeartbeatResponse ver="1.0" timestamp="2018-06-26T04:28:01.000+0000">
    <Error errMsg="OK" errCode="200"/>
    <Status>UP</Status>
</HeartbeatResponse>
